# 8. BIBLIOGRAFÍA Y WEBGRAFÍA

Las fuentes utilizadas durante el desarrollo de PacePal se agrupan en tres bloques: documentación interna generada por el equipo a lo largo de los sprints, material docente proporcionado por el centro, y documentación oficial de las tecnologías y herramientas empleadas en el proyecto.

---

## Documentación interna del proyecto PacePal

- Definición inicial del proyecto y propuesta de valor — elaborada en Sprint 0.
- Documentación Scrum base: roles, reglas de evidencias y cadencia de sprints.
- Backlog de historias de usuario con criterios de aceptación y estimaciones.
- Guía de estilos: paleta cromática, tipografía, iconografía y componentes de interfaz.
- Wireframes desktop y mobile de todas las pantallas principales.
- Informe de accesibilidad formal: Lighthouse, WAVE y WCAG Contrast Checker.
- Inspección de usabilidad con revisión heurística y prueba con usuario externo.
- Documentación de la API REST: endpoints, métodos, parámetros y respuestas JSON.
- Pruebas funcionales por flujo y documentación de resultados obtenidos.
- Colección de pruebas de la API con Postman.
- Pruebas automatizadas básicas con Selenium WebDriver.
- Guía de instalación local con XAMPP, Apache, phpMyAdmin y Node.js.
- Esquema SQL de la base de datos y modelo relacional.
- Documentación del cliente React: decisión de migración, fallback demo y publicación en GitHub Pages.
- README técnico del proyecto con instrucciones de arranque.

---

## Material docente y enunciados del proyecto

- Plantilla oficial de estructura de la memoria del Proyecto Agile Intermodular.
- Rúbrica de evaluación del Proyecto Agile Intermodular 2025-2026.
- Programación del Proyecto Agile Intermodular 25-26 — documento de referencia del centro.
- Materiales del módulo DIW (Sprints 1, 2 y 3) — guías de diseño, accesibilidad y usabilidad.
- Materiales del módulo DWEC (Sprints 1, 2 y 3) — formularios, validaciones, AJAX y React.
- Materiales del módulo DWES Sprint 2 — API REST, PDO, sesiones y roles.
- Materiales de despliegue Sprint 3 — publicación estática, entorno local y GitHub Pages.

## Documentación oficial y recursos externos

- **MDN Web Docs** — Referencia principal de HTML, CSS, JavaScript, Fetch API, formularios y técnicas de accesibilidad.
  https://developer.mozilla.org

- **PHP Manual** — Documentación oficial de PHP sobre sesiones, PDO, consultas preparadas y respuestas JSON.
  https://www.php.net/docs.php

- **React** — Documentación oficial de componentes, hooks, contexto y gestión de estado.
  https://react.dev

- **Vite** — Documentación de configuración, entorno de desarrollo y build del cliente.
  https://vitejs.dev/guide/

- **Bootstrap 5** — Referencia de grid, utilidades, componentes y alertas usados en la interfaz.
  https://getbootstrap.com/docs/5.3/

- **Bootstrap Icons** — Catálogo de iconografía utilizada en navegación, botones y tarjetas.
  https://icons.getbootstrap.com

- **W3C / WAI — WCAG 2.1** — Criterios de accesibilidad usados como referencia en la auditoría DIW.
  https://www.w3.org/WAI/WCAG21/quickref/

- **Google Lighthouse** — Herramienta de auditoría de accesibilidad, rendimiento, SEO y buenas prácticas.
  https://developer.chrome.com/docs/lighthouse/

- **WAVE / WebAIM** — Herramienta de análisis de accesibilidad usada para detectar errores y alertas en la landing.
  https://wave.webaim.org

- **MySQL Documentation** — Referencia de SQL y modelo relacional para el diseño de la base de datos.
  https://dev.mysql.com/doc/

- **GitHub Pages** — Documentación del servicio de publicación estática usado para desplegar el cliente React.
  https://docs.github.com/en/pages

- **Postman Learning Center** — Guía de colecciones, entornos y pruebas de API REST.
  https://learning.postman.com
