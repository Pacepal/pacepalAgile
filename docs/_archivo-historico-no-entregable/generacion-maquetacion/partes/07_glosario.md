# 7. GLOSARIO

Este glosario recoge los principales términos técnicos utilizados en la memoria de PacePal. Las definiciones están adaptadas al contexto del proyecto y están pensadas para que la memoria sea autocontenida y comprensible sin necesidad de consultar fuentes externas.

| Término                 | Definición en el contexto de PacePal                                                                                                                                                                                                                        |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Accesibilidad**       | Conjunto de criterios y técnicas aplicadas para que la interfaz sea usable por el mayor número posible de personas, incluyendo usuarios con diversidad funcional. En PacePal se auditó formalmente la landing con Lighthouse, WAVE y WCAG Contrast Checker. |
| **AJAX**                | Técnica de petición asíncrona que permite cargar o filtrar información sin recargar la página completa. Se aplica en el buscador dinámico de productos de la tienda.                                                                                        |
| **API REST**            | Interfaz de programación que expone rutas HTTP para acceder y manipular los recursos del sistema (productos, rutas, actividades, sesión, pedidos, usuarios y cookies) devolviendo respuestas en JSON. En PacePal la implementa el backend PHP.              |
| **Backend**             | Parte del sistema que ejecuta PHP, accede a la base de datos y gestiona las sesiones de usuario. No es directamente visible por el usuario final; el cliente React se comunica con él a través de la API REST.                                              |
| **Backlog**             | Lista priorizada de historias de usuario y tareas del proyecto, gestionada por el Product Owner y actualizada a lo largo de los sprints. El backlog completo de PacePal se recoge en el Anexo 9.6.                                                          |
| **Bootstrap**           | Framework de CSS y componentes de interfaz usado en PacePal para utilidades de layout, tablas, espaciado, alertas y apoyo visual en formularios. Se usa junto a Bootstrap Icons para la iconografía.                                                        |
| **Carrito**             | Funcionalidad de la tienda que permite al usuario añadir productos, modificar cantidades, eliminar artículos y generar un pedido. En PacePal el estado del carrito se gestiona en el cliente React.                                                         |
| **CRUD**                | Conjunto de operaciones básicas sobre datos: Create (crear), Read (leer), Update (actualizar) y Delete (eliminar). Implementadas en los controladores del backend para cada entidad del sistema.                                                            |
| **Endpoint**            | Ruta concreta de la API que acepta un método HTTP y devuelve una respuesta JSON. Por ejemplo: `/api/login`, `/api/productos` o `/api/actividades`.                                                                                                          |
| **Fallback demo**       | Comportamiento alternativo del cliente React cuando el backend PHP no está disponible. En PacePal, el cliente publicado en GitHub Pages usa respuestas JSON estáticas para simular la interacción sin servidor real.                                        |
| **Frontend**            | Parte del sistema que el usuario ve e con la que interactúa directamente. En PacePal está implementada en HTML, CSS, JavaScript y React con Vite.                                                                                                           |
| **GitHub Pages**        | Servicio de hosting estático de GitHub usado para publicar el cliente React compilado de PacePal. No admite PHP ni MySQL, por lo que requiere el fallback demo documentado.                                                                                 |
| **Historia de usuario** | Descripción breve de una funcionalidad desde la perspectiva del usuario, con el formato "Como... quiero... para poder...", criterios de aceptación y estimación de esfuerzo. Base del backlog de PacePal.                                                   |
| **JSON**                | Formato de texto ligero para el intercambio de datos entre el cliente y el servidor. Todas las respuestas de la API REST de PacePal son JSON. También es el formato base del fallback demo.                                                                 |
| **Lighthouse**          | Herramienta de auditoría integrada en Chrome DevTools que evalúa rendimiento, accesibilidad, SEO y buenas prácticas de una página web. PacePal obtuvo puntuación 100 en accesibilidad en la auditoría de la landing.                                        |
| **MySQL / MariaDB**     | Sistema de gestión de base de datos relacional usado para persistir los datos de PacePal. Se gestiona en local mediante XAMPP y phpMyAdmin.                                                                                                                 |
| **PDO**                 | PHP Data Objects; capa de abstracción para el acceso a base de datos en PHP que permite usar consultas preparadas y previene la inyección SQL. Usado en todos los modelos del backend de PacePal.                                                           |
| **Postman**             | Herramienta para probar y documentar APIs. Se usó en PacePal para validar los endpoints de la API REST mediante una colección de pruebas con entorno local.                                                                                                 |
| **React**               | Librería de JavaScript usada en el Sprint 3 para construir el cliente de PacePal como una SPA basada en componentes reutilizables, hooks y servicios asíncronos.                                                                                            |
| **Responsive**          | Capacidad de la interfaz para adaptarse correctamente a distintos tamaños de pantalla. En PacePal se implementa de forma modular con hojas CSS independientes por sección.                                                                                  |
| **Rol**                 | Nivel de permisos asignado a un usuario dentro del sistema. PacePal diferencia tres roles: invitado, usuario registrado y administrador.                                                                                                                    |
| **Scrum**               | Marco de trabajo ágil adoptado de forma educativa en PacePal, con backlog priorizado, sprints delimitados, revisión de entregables y retrospectivas simplificadas.                                                                                          |
| **Sesión**              | Estado de autenticación del usuario mantenido en el servidor PHP mediante `$_SESSION`. En el fallback demo de GitHub Pages se simula mediante lógica en el cliente.                                                                                         |
| **SPA**                 | Single Page Application; tipo de aplicación web que carga una única página HTML y actualiza el contenido dinámicamente sin recargas completas. El cliente React de PacePal sigue este modelo.                                                               |
| **Sprint**              | Período de trabajo delimitado con objetivos definidos, revisión de resultados y actualización del backlog. PacePal se desarrolló en cuatro sprints más una fase de cierre documental.                                                                       |
| **Vite**                | Herramienta de desarrollo y build para proyectos JavaScript modernos. Usada en PacePal para desarrollar y compilar el cliente React con recarga en caliente y build optimizado.                                                                             |
| **WAVE**                | Herramienta de análisis de accesibilidad de WebAIM que detecta errores y alertas en el HTML de una página. En la auditoría de la landing de PacePal se obtuvieron 0 errores.                                                                                |
| **WCAG**                | Web Content Accessibility Guidelines; estándar internacional de accesibilidad del W3C. Usado como referencia en la auditoría de accesibilidad DIW de PacePal, nivel objetivo A.                                                                             |
| **XAMPP**               | Paquete de software que integra Apache, PHP, MySQL/MariaDB y phpMyAdmin para ejecutar aplicaciones web en local. Entorno principal de desarrollo y despliegue local de PacePal.                                                                             |
